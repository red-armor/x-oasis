CRITICAL FINDINGS:
==================

1. @x-oasis/async-call-rpc-web is ONLY used in:
   - createPageBridge.ts (line 28: require('@x-oasis/async-call-rpc-web').default)

2. Usage Pattern:
   - Dynamic require at runtime
   - Only when createPageBridge() is called
   - NOT a static import
   - NOT declared in package.json dependencies or peerDependencies

3. Why Not a Hard Dependency:
   - Optional feature (only for page bridge scenarios)
   - Keeps bundle sizes small for other use cases
   - Testable without npm installation (via vitest alias)

4. How It Works in createPageBridge:
   - Requires RPCMessageChannel from async-call-rpc-web
   - Creates instance with description
   - Binds MessagePort via orchestrator handler
   - Exposes send/onMessage API via contextBridge

5. Error Handling:
   - Try-catch around require
   - Helpful error message if package not installed
   - Falls back to globalThis if contextBridge unavailable
