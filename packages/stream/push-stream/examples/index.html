@x-oasis/push-stream Visual Examples

Single-consumer push-pull async stream with enqueue/done/error semantics

Modeled after claude-code's Stream<T>. Key differences from @x-oasis/event-stream: no isComplete / extractResult, strict single-consumer enforcement, and explicit done() termination.

Basic Enqueue / Done

Producer calls enqueue(value) to push values. Values buffer when no consumer is listening. Consumer pulls them via for await...of. Producer calls done() to terminate. Watch values flow from left to right.

Unlike EventStream, PushStream enforces strict single-consumer semantics. Calling [Symbol.asyncIterator]() a second time throws an error. Click "Run Demo" to see the first consumer succeed and the second one fail.

LLM Token Streaming

Simulates an LLM producing tokens over time. The producer enqueue()s each token and calls done() when finished. The consumer renders tokens as they arrive. This mirrors claude-code's streaming pattern.

Error Handling

Demonstrates two termination paths: stream.done() ends iteration normally, while stream.error() rejects the pending next() call. Subsequent next() calls after error() also reject with the stored error.

Early Break & onReturn

When the consumer breaks out of for await...of early, the iterator's return() method is called, which marks the stream as done and invokes the onReturn callback. This allows the producer to clean up resources (e.g. abort a fetch).
